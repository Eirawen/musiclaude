# Blind A/B/C Listening Experiment — Results

**Date:** 2026-03-16
**Evaluator:** Khaled (blind — did not know which condition produced which track)

## Design

3 songs, 3 conditions each, randomized into 9 numbered MP3s.

- **Songs:** Piano Waltz, Violin-Piano Duet, Piano Prelude
- **Conditions:**
  - **Baseline** — Raw first draft, no feedback loop
  - **XGBoost** — Revised using XGBoost predicted rating + feature deficiencies
  - **Profile** — Revised using percentile-based feature profile (ranked by importance)

All compositions generated by Claude (Opus) from the same song prompts. Feedback conditions used
the same baseline as starting point, with one round of revision guided by the respective feedback system.

**Averages:** Baseline 75.3, XGBoost 76.7 (+1.3), Profile 82.3 (+7.0)

## Key Findings

1. **Profile wins 2/3 songs** and produced the overall favorite (waltz at 93/100).
2. **Both feedback methods beat baseline in every song.** The feedback loop works.
3. **Profile improves scores 5x more than XGBoost** on average (+7.0 vs +1.3).
4. **XGBoost hurt the waltz** (82 vs 88 baseline) — the evaluator thought the XGBoost version was the *unrevised baseline*, calling it "simplest, least expressive."
5. **Profile pieces praised for subtlety and style** — evaluator consistently noted "actual style", "subtle touches", "felt like a human being playing it."
6. **Evaluator correctly identified all 3 song families** blind (grouped variants together).

## Conclusion

Feature profile feedback (percentile-based, ranked by importance) produces substantially better compositions than XGBoost score-based feedback. The profile system tells Claude *what specifically to improve* with concrete targets, while XGBoost gives blunt signals that can lead to over-correction or regression. Profile is now the default feedback pathway.
